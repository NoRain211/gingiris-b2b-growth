# 파트 2: 성장 엔진 구축

## 1. PLG + SLG 하이브리드 모델

| 모델 | 정의 | 장점 |
|:---|:---|:---|
| **PLG** | 제품 자체를 핵심 마케팅 도구로 활용 | 극도로 낮은 CAC, 높은 확장성 |
| **SLG** | 영업팀 주도 성장 | 복잡한 딜 클로징, 높은 LTV |

성공적인 B2B 기업 대부분은 하나만 선택하지 않는다 — 둘 다 동시에 운영한다. PLG는 퍼널 상단을 대량의 리드로 채우고, SLG는 셀프서브로는 클로징할 수 없는 고가치 어카운트를 확보한다. 진짜 어려운 부분은 둘 사이의 핸드오프다.

### 전환율 벤치마크

| 모션 | 지표 | 일반 범위 | 상위 25% |
|:---|:---|:---|:---|
| **Freemium → 유료** | 무료→유료 전환율 | 2–5% | 7–10% |
| **Free Trial (카드 미등록)** | 트라이얼→유료 전환율 | 8–15% | 18–25% |
| **Free Trial (카드 등록)** | 트라이얼→유료 전환율 | 30–50% | 55–65% |
| **Reverse Trial** | 다운그레이드 방지율 | 40–60% | 65%+ |
| **PQL → SQL** | 영업 수용률 | 20–35% | 40–50% |
| **SQL → Closed Won** | 수주율 (PLG 소싱) | 25–40% | 45%+ |
| **PLG 소싱 ACV** | 평균 계약 금액 | 아웃바운드보다 낮음 | 확장 후 동등 수준 |

Reverse Trial(N일간 전체 기능을 제공한 후 무료로 다운그레이드)은 순수 프리미엄이나 일반 무료 체험보다 일관되게 더 높은 성과를 보인다. Notion, Airtable, Figma 모두 이 모델로 전환했다.

### PQL (Product Qualified Lead) 스코어링 모델

PQL은 기존 MQL을 대체한다. 마케팅 참여도(백서 다운로드 여부)를 측정하는 대신, 제품 참여도(실제로 제품을 사용하는지)를 측정한다. 이것이 PLG의 핵심 장점이다 — 영업팀이 연락하기 전에 이미 누가 가치를 얻고 있는지 알 수 있다.

#### 스코어링 차원 및 가중치

| 차원 | 시그널 | 점수 | 가중치 | 근거 |
|:---|:---|:---|:---|:---|
| **활성화** | 핵심 가치 행동 완료 (예: 비디오 3건 이상 내보내기, 첫 워크플로우 생성) | 0–25 | 높음 | 제품의 핵심 가치를 실제로 경험했음을 증명 |
| **빈도** | 14일 롤링 기준 로그인 빈도 | 0–20 | 높음 | 매일/매주 사용 = 워크플로우에 통합됨 |
| **깊이** | 핵심 행동 이외에 사용한 기능 수 | 0–15 | 중간 | 폭넓은 채택은 제품 고착도를 의미 |
| **협업** | 3명 이상의 팀원 초대, 공유 워크스페이스 생성 | 0–20 | 높음 | 다중 사용자 = 조직 차원의 도입, 이탈이 훨씬 어려워짐 |
| **기업 적합성** | 회사 규모, 산업, 도메인 (Fortune 500 이메일 vs. gmail) | 0–10 | 중간 | 지불 능력 ≠ 지불 의향이지만, 여전히 중요 |
| **확장 시그널** | 사용량 한도 도달, 가격 페이지 조회, API 사용 | 0–10 | 중간 | 행동 데이터를 보완하는 구매 의향 시그널 |

**총점: 100점**

#### PQL 임계값

| 점수 범위 | 분류 | 조치 |
|:---|:---|:---|
| **0–30** | 너처링 | 인앱 교육, 온보딩 이메일, 영업 접촉 없음 |
| **31–55** | Warm PQL | 자동화된 넛지, 인앱 업그레이드 프롬프트, 로우터치 영업 (풀형 CSM) |
| **56–75** | Hot PQL | 24시간 이내 SDR 아웃리치, 사용 데이터 기반 개인화 |
| **76–100** | 영업 긴급 | 4시간 이내 AE 직접 연락, 화이트 글러브 대응 |

#### 영업 아웃리치를 위한 행동 트리거

점수 임계값만 기다리지 말 것 — 아래 특정 이벤트는 누적 점수에 관계없이 즉각적인 영업 조치를 트리거해야 한다:

| 트리거 | 중요한 이유 | 대응 시간 |
|:---|:---|:---|
| **7일 내 가격 페이지 3회 이상 방문** | 적극적인 구매 검토 중 | 4시간 이내 SDR 이메일 |
| **사용량 한도 도달** | 무료 티어 이상이 필요 | 인앱 업그레이드 프롬프트 + 24시간 이내 SDR 아웃리치 |
| **관리자가 한 세션에서 5명 이상 초대** | 조직 차원의 롤아웃 시작 | 당일 업무 시간 내 AE 전화 |
| **Enterprise SSO 또는 SCIM 설정 시도** | IT/보안 검토 진행 중 | 4시간 이내 SE + AE 공동 아웃리치 |
| **API 키 생성 + 첫 API 호출** | 개발자 연동 시작 | 24시간 이내 Developer Relations + AE |
| **데이터 내보내기 또는 다운로드** | 경쟁사 평가 중이거나 내부 발표 준비 중일 수 있음 | 24시간 이내 SDR 아웃리치 |
| **최초 가입자가 아닌 사용자가 결제 페이지 방문** | 재무/구매 부서 관여 | 4시간 이내 AE 아웃리치 (ROI 자료 포함) |

### PLG → SLG 핸드오프 프로세스

대부분의 하이브리드 모델이 무너지는 지점이 바로 여기다. 영업팀이 영업과 이야기하고 싶지 않은 셀프서브 사용자에게 연락하거나, 더 심각하게는, 구매에 절실하게 도움이 필요한 사용자에게 영업팀이 전혀 연락하지 않는 경우다.

#### 핸드오프 워크플로우

**1단계: PQL 식별 (자동화)**
- 프로덕트 분석 플랫폼(Amplitude, Mixpanel, Heap)이 사용 데이터를 데이터 웨어하우스에 전달
- 스코어링 모델이 스케줄에 따라 실행 (Hot 트리거는 매시간, 누적 점수는 매일)
- PQL이 전체 사용 컨텍스트와 함께 CRM(Salesforce, HubSpot)에 푸시됨

**2단계: 데이터 보강 (자동화)**
- PQL 생성 시 Clearbit/ZoomInfo 보강 자동 실행
- 회사 규모, 산업, 기술 스택, 펀딩 단계 정보 추가
- 기존 어카운트 매칭: 이 사용자가 이미 파이프라인에 있는 회사 소속인가?

**3단계: 라우팅**
- 점수 56–75: SDR 팀 라운드 로빈 배정
- 점수 76+: 이미 담당 AE가 있으면 해당 AE로 라우팅, 없으면 SDR 팀 리드에게 배정 요청
- 엔터프라이즈 도메인 감지: 점수와 무관하게 엔터프라이즈 AE로 라우팅
- 기존 고객 (확장): 담당 CSM으로 라우팅

**4단계: 컨텍스트 기반 아웃리치**
영업 담당자는 반드시 사용자의 실제 제품 사용 내역을 참조해야 한다. PQL에게 보내는 일반적인 아웃리치는 아무 아웃리치도 하지 않는 것보다 나쁘다.

나쁜 예: "안녕하세요, [제품]에 가입하신 것을 확인했습니다. 데모 일정을 잡아볼까요?"
좋은 예: "안녕하세요, 팀에서 이번 주에 12개 워크플로우를 만드셨고 현재 플랜의 50개 워크플로우 한도에 가까워지고 있는 것을 확인했습니다. 이 단계의 팀은 보통 [유료 플랜의 특정 기능]이 필요합니다. [유사 기업]이 이 문제를 어떻게 해결했는지 설명해 드릴까요?"

**5단계: 피드백 루프**
- 영업이 CRM에 PQL 결과 기록: 전환됨, 부적격(사유 포함), 미준비(너처링으로 재순환)
- 데이터 팀이 PQL→SQL 전환율을 월별로 검토
- 실제 전환 데이터 기반으로 분기별 스코어링 모델 재보정

#### 일반적인 실패 유형

| 실패 유형 | 증상 | 해결책 |
|:---|:---|:---|
| 영업이 PQL을 무시 | 낮은 접촉률, PQL 기한 만료 | PQL 팔로업을 추적 KPI로 설정, PQL 수주율이 아웃바운드의 2–3배임을 영업에게 제시 |
| 영업이 너무 일찍 접촉 | 높은 옵트아웃율, 영업 접촉에 대한 부정적 NPS | PQL 임계값 상향, 필수 활성화 기준 추가 |
| CRM에 사용 컨텍스트 없음 | 담당자가 일반적인 아웃리치 발송 | 어카운트별 사용 현황 대시보드를 구축하여 CRM에 임베드 |
| PLG팀과 SLG팀의 경쟁 | 기여도 귀속을 둘러싼 채널 갈등 | 명확한 규칙: 영업 접촉 전에 셀프서브한 경우 PLG에 크레딧 부여. 보상 체계가 어느 채널도 불이익을 주지 않도록 설계 |
| 무료 티어가 너무 관대 | 낮은 전환율, 높은 인프라 비용 | 프리미엄 기능의 기간 한정 체험, 특히 협업 기능에 대한 사용량 제한 |

## 2. 어필리에이트 마케팅

| 기업 | 모델 | 쿠키 기간 | 적합한 대상 |
|:---|:---|:---|:---|
| **InVideo** | **첫 주문의 50%** | 120일 | 중저가, 높은 전환율의 도구 |
| **HeyGen** | **12개월간 20%** | 90일 | 구독 기반, 높은 리텐션 SaaS |
| **Deel** | **$500–$1500 고정 바운티** | 90일 | 고단가 엔터프라이즈 서비스 |

### 커미션 모델 선택

위 세 가지 모델은 근본적으로 다른 전략적 판단을 반영한다:

**첫 주문 대비 비율**은 제품 가격이 낮고 트라이얼→유료 전환율이 높을 때 효과적이다. 볼륨에 투자하는 것이다. 어필리에이트의 인센티브는 최대한 많은 가입을 유도하는 데 맞춰진다.

**반복 비율**은 제품의 리텐션이 강할 때 효과적이다. 어필리에이트의 인센티브를 자사 인센티브와 일치시키는 것이다 — 어필리에이트는 오래 머무는 고객을 추천해야 이득이 된다. 대부분의 B2B SaaS에 가장 적합한 모델이다.

**고정 바운티**는 ACV가 높은 제품에 적합하다. 비율 기반 커미션이 비현실적으로 높아질 수 있는 경우(사기 유인이 됨)나, 영업 사이클이 길어서 어필리에이트가 엔터프라이즈 리드 추천에 동기를 가져야 하는 경우에 사용한다.

### 최적 커미션 비율 계산

계산 방법:

```
최대 커미션 = (고객 LTV × 목표 이익률 잔여분) - 기타 CAC 구성요소

예시:
- 평균 고객 LTV: $6,000 (3년간)
- CAC 차감 후 목표 혼합 매출총이익률: 70%
- 고객 확보에 사용 가능한 예산: $6,000 × 30% = $1,800
- 기타 CAC 구성요소 (온보딩, 도구, 어트리뷰션 플랫폼 수수료): $200
- 최대 어필리에이트 커미션: $1,600

반복 지급 시: $1,600 ÷ 36개월 = ~$44/월
평균 MRR이 $167/월이면, 이는 ~26% 반복 커미션에 해당
첫 주문만: $167 첫 달에 $1,600은 958%로, 비현실적
  → 따라서 첫 주문 %의 경우, 일부만 적용: 예를 들어 첫 결제의 30–50% ($50–$83)
```

**경험 법칙:**
- 반복 커미션: MRR의 15–30%, 12–24개월간
- 첫 주문 커미션: 첫 달 결제 금액의 30–100%
- 고정 바운티: 첫 해 ACV의 10–25%
- 어필리에이트 소싱 고객의 페이백 기간이 항상 12개월 이내가 되도록 할 것
- 보수적으로 시작할 것 — 커미션을 올리기는 쉽지만 내리기는 고통스럽다

### 어필리에이트 리크루팅 전략

#### 어필리에이트를 찾을 수 있는 곳

| 채널 | 어필리에이트 유형 | 예상 품질 | 볼륨 |
|:---|:---|:---|:---|
| **자사 고객 기반** | 이미 적극 추천 중인 파워유저 | 최고 (진정성 있는 옹호) | 낮음–중간 |
| **해당 카테고리 YouTube 리뷰어** | 튜토리얼 채널의 콘텐츠 크리에이터 | 높음 (타겟 오디언스) | 중간 |
| **비교/리뷰 사이트** (G2, Capterra 어필리에이트) | 전문 리뷰어 | 중간 (실리적이지만 효과적) | 높음 |
| **업계 블로거 및 뉴스레터 운영자** | 니치 권위 사이트 | 높음 (독자에게 신뢰받음) | 낮음 |

핵심 포인트: 진정성 있게 개인화하고, 상대방의 콘텐츠 품질을 먼저 인정하며, 수익 잠재력을 구체적으로 제시하고, 제품 액세스를 제안한다.

### 성과 티어 및 인센티브 구조

| 티어 | 월간 추천 건수 | 커미션 비율 | 보너스 혜택 |
|:---|:---|:---|:---|
| **Standard** | 1–10 | 기본 비율 (예: 20%) | 어필리에이트 대시보드, 기본 크리에이티브 |
| **Silver** | 11–25 | 기본 + 5% 보너스 | 전담 어필리에이트 매니저, 커스텀 랜딩 페이지 |
| **Gold** | 26–75 | 기본 + 10% 보너스 | 신기능 얼리 액세스, 코브랜드 콘텐츠, 분기 보너스 |
| **Platinum** | 76+ | 기본 + 15% 보너스 | 커스텀 커미션 협상, 공동 웨비나, 연간 리트릿 초대, 제품 자문위원회 참여 |

효과적인 가속 장치:
- **런칭 보너스**: 신규 어필리에이트 가입 후 첫 90일간 2배 커미션
- **시즌별 푸시**: 비수기에 보너스 커미션을 적용해 매출 변동 완화
- **SPIF** (Sales Performance Incentive Fund): 특정 기간 내 스트레치 목표 달성 시 일회성 보너스
- **티어 소급 적용**: 임계값 도달 시 해당 월 전체 추천 건에 소급하여 상위 커미션 적용 (티어 경계를 넘기려는 동기 부여)

### 어필리에이트 리소스 키트

어필리에이트가 브랜드에 맞지 않거나 부정확한 콘텐츠를 만들지 않도록 필요한 모든 것을 제공한다:

- **크리에이티브**: 배너 광고 (표준 IAB 사이즈), 소셜 미디어 이미지, 이메일 템플릿
- **카피**: 사전 작성된 제품 설명 (짧은/중간/긴 버전), 핵심 차별화 포인트, 경쟁사 비교 대화 포인트
- **랜딩 페이지**: 일반 홈페이지보다 전환율이 높은 코브랜드 커스텀 랜딩 페이지
- **데모 비디오**: 어필리에이트가 임베드하거나 링크할 수 있는 짧은 제품 워크스루
- **데이터 시트**: ROI 계산기, 케이스 스터디 요약, 산업 통계
- **트래킹 링크**: 고유 어필리에이트 ID가 포함된 UTM 태그 링크, 특정 페이지/기능으로의 딥링크
- **할인 코드**: 어필리에이트 오디언스 전용 코드 (어트리뷰션에도 유용)
- **콘텐츠 캘린더**: 어필리에이트가 콘텐츠를 맞출 수 있는 예정된 출시, 기능, 프로모션 일정

### 어트리뷰션 모델

| 모델 | 작동 방식 | 적합한 경우 | 리스크 |
|:---|:---|:---|:---|
| **Last-click** | 전환 직전 마지막 클릭한 어필리에이트 링크에 크레딧 부여 | 단순한 프로그램, 로우터치 제품 | 인지도를 높인 퍼널 상단 어필리에이트 무시 |
| **First-click** | 처음 클릭한 어필리에이트 링크에 크레딧 부여 | 인지도/발견 보상 | 딜을 클로징한 어필리에이트 무시 |
| **Linear** | 모든 어필리에이트 접점에 균등 분배 | 공정하지만 복잡 | 커미션 희석, 이해하기 어려움 |
| **Time-decay** | 전환에 가까운 접점에 더 많은 크레딧 | 균형 잡힌 접근 | 정교한 트래킹 필요 |
| **Last-click + 룩백 윈도우** | 쿠키 유효 기간 내 마지막 클릭 | 대부분의 B2B SaaS 프로그램 (실용적 기본값) | 윈도우 길이가 중요: 너무 짧으면 크레딧 누락, 너무 길면 과잉 귀속 |

대부분의 B2B SaaS에는 **60–90일 룩백 윈도우를 적용한 Last-click** 모델을 사용한다. 어필리에이트가 이해하기 충분히 단순하고, B2B 구매 검토 주기를 포착할 만큼 충분히 긴 윈도우다.

### 어필리에이트 사기 탐지

사기는 반드시 발생한다. 미리 대비해야 한다.

| 사기 유형 | 수법 | 탐지 방법 | 예방 |
|:---|:---|:---|:---|
| **Cookie stuffing** | 어필리에이트가 고트래픽 사이트의 숨겨진 iframe/이미지를 통해 쿠키를 심어 오가닉 전환에 대한 크레딧을 주장 | 비정상적으로 높은 노출 대비 클릭 비율, 실제 클릭 없는 전환 | 서버사이드 트래킹 사용, 실제 클릭 이벤트 필수화, 클릭-전환 상관관계 모니터링 |
| **Self-referral** | 어필리에이트가 자신의 링크로 가입 | 어필리에이트 이메일/결제 정보와 고객 기록 대조 | 이용약관에 명시적으로 금지, 동일 도메인/IP 자동 플래그, 어필리에이트 가입과 추천 고객 간 최소 30일 간격 필수 |
| **Click fraud** | 봇이나 클릭 팜이 가짜 클릭을 생성하여 지표 부풀리기 (CPC 모델에 해당) | 트래픽 분석: 불가능한 지역 패턴, 균일한 클릭 간 시간, 비정상적 이탈률 확인 | 클릭 기반이 아닌 전환 기반으로 지급. CPC를 사용해야 하면 최소 세션 지속 시간 요구 |
| **상표 입찰** | 어필리에이트가 자사 브랜드명으로 Google Ads를 구매하여 이미 자사를 검색하는 사람에 대한 크레딧 주장 | SEMrush나 SpyFu 등의 도구로 브랜드 SEM 모니터링 | 어필리에이트 계약에 브랜드 입찰을 명시적으로 금지, 정기 감사로 시행 |
| **인센티브 가입** | 어필리에이트가 사람들에게 자신의 링크로 가입하면 돈을 지급 | 매우 높은 가입 볼륨에 거의 제로에 가까운 활성화/참여도 | 커미션 트리거 조건을 가입이 아닌 활성화 마일스톤으로 설정 (가입이 아닌 첫 가치 행동) |
| **리드 재활용** | 동일 리드를 여러 채널에 제출하거나 만료된 리드를 재제출 | 이메일, 전화번호, 회사 도메인 기반 중복 탐지 | 기존 파이프라인과 중복 제거, 90일 제외 기간 적용 |

**예상 사기 기본 비율:** 어필리에이트 기반 전환의 5–15%는 사기성이거나 저품질이다. 이를 재무 모델에 반영하고 적극적으로 감사한다. 지급 전 30–60일 보류 기간(클로백 윈도우)을 운영한다.

## 3. 채널 파트너 에코시스템

### 4가지 핵심 파트너 유형

| 파트너 유형 | 핵심 가치 | 협업 모델 |
|:---|:---|:---|
| **기술 통합 파트너** | 제품 기능 확장 | API 연동, 마켓플레이스 등록 |
| **솔루션 파트너** | 산업별 솔루션 및 구현 | 컨설팅, 구현, 리셀 |
| **클라우드 마켓플레이스** | 클라우드 대기업 채널 활용 | AWS/Azure/GCP Marketplace 등록 |
| **전략적 제휴** | 브랜드 보증 및 공동 판매 | 공동 마케팅, 공동 판매, 공동 개발 |

### 파트너 리크루팅 플레이북

#### 잠재 파트너 식별

아래 조건에 해당하는 기업을 찾는다:
- 자사 ICP에 판매하지만 경쟁하지 않는 기업
- 제품 차별화가 필요한 서비스 사업을 보유한 기업
- 이미 비공식적으로 자사와 연동 중인 기업 (API 로그를 확인)
- "저희를 어떻게 알게 되셨나요?" 응답에 등장하는 기업
- 고객이 자신의 기술 스택의 일부로 언급하는 기업

#### 리크루팅 프로세스

**1단계: 리서치 및 타겟팅 (1–2주차)**
- 50–100개 잠재 파트너 대상 목록 구축
- 평가 기준: 자사 ICP와의 오디언스 중복도, 기술 역량, 시장 영향력, 전략적 정합성
- 적절한 담당자 파악: 파트너십/BD 팀 리더, CEO가 아님 (소규모 기업 제외)

**2단계: 아웃리치 (3–4주차)**
- 자사가 필요한 것이 아닌, 상대방에게 해줄 수 있는 것을 먼저 제시
- 데이터 제시: "자사 제품과 [상대방 제품]을 함께 사용하는 고객의 리텐션이 40% 더 높습니다"
- 저부담 파일럿 제안: 공동 웨비나, 연동 프로젝트, 공동 저술 콘텐츠

**3단계: 파일럿 (5–12주차)**
- 사전에 성공 지표 정의: 공동 고객 수, 영향받은 매출, 연동 채택률
- 전담 파트너 매니저 배정 (공유 리소스가 아닌)
- 단일 캠페인 또는 세그먼트에서 공동 GTM 모션 실행

**4단계: 공식화 (13주차 이후)**
- 파일럿이 지표를 달성하면 파트너 계약으로 공식화
- 티어 배정, 교육 요건, 매출 커미트먼트 정의
- 파트너십 공개 발표 (보도 자료, 공동 블로그, 마켓플레이스 등록)

### 파트너 티어 및 요건

| 티어 | 연간 매출 요건 | 인증 담당자 수 | 요건 | 혜택 |
|:---|:---|:---|:---|:---|
| **Registered** | $0 (신규 파트너) | 0 | 파트너 계약 체결, 온보딩 완료 | 파트너 포털 액세스, 기본 교육, 10% 추천 수수료 |
| **Silver** | $50K 영향 매출 | 2 | 영업 인증 완료, 1건 공동 고객 수주 | 15% 추천/리셀 마진, 코마케팅 펀드 ($2K/분기), 파트너 디렉토리 등록 |
| **Gold** | $200K 영향 매출 | 5 | 기술 인증 완료, 분기별 비즈니스 리뷰, 전담 고객 성공 | 20% 추천/리셀 마진, 코마케팅 펀드 ($10K/분기), 공동 케이스 스터디, 우선 지원, 베타 액세스 |
| **Platinum** | $500K+ 영향 매출 | 10+ | 경영진 스폰서 정렬, 연간 계획, 공동 제품 로드맵 의견 반영 | 25%+ 마진 (협상), $25K+/분기 MDF, 자사 AE와 공동 판매, 파트너 자문위원회, 경영진 브리핑 |

이 수치는 ACV $50K–$200K의 SaaS 제품 기준으로 산정되었다. 자사 가격대에 비례하여 임계값을 조정한다.

### 파트너 인에이블먼트 프로그램

#### 인증 커리큘럼

**영업 인증 (Silver 이상 필수)**
- 모듈 1: 제품 개요 및 포지셔닝 (2시간)
- 모듈 2: ICP 및 바이어 페르소나 (1시간)
- 모듈 3: 경쟁 환경 및 반론 처리 (2시간)
- 모듈 4: 데모 진행 (2시간 + 실기 시험)
- 모듈 5: 가격, 패키지, 딜 구성 (1시간)
- 평가: 자사 팀에게 모의 데모를 진행하여 80% 이상으로 통과
- 재인증: 연 1회, 신기능 업데이트 모듈 포함

**기술 인증 (Gold 이상 필수)**
- 모듈 1: 아키텍처 및 연동 패턴 (4시간)
- 모듈 2: 구현 방법론 (4시간)
- 모듈 3: API 심층 분석 및 커스텀 개발 (4시간)
- 모듈 4: 보안, 컴플라이언스, 데이터 처리 (2시간)
- 모듈 5: 트러블슈팅 및 에스컬레이션 절차 (2시간)
- 평가: 레퍼런스 구현 완료, 기술 리뷰 통과
- 재인증: 연 1회

**딜리버리 인증 (선택, 구현 파트너용)**
- 모듈 1: 프로젝트 스코핑 및 SOW 개발 (2시간)
- 모듈 2: 마이그레이션 및 데이터 온보딩 플레이북 (4시간)
- 모듈 3: 최종 사용자를 위한 변경 관리 (2시간)
- 모듈 4: 고라이브 이후 성공 측정 (1시간)
- 평가: 감독 하에 구현 리드

#### 파트너 영업 교육

파트너에게 자사 영업팀이 사용하는 것과 동일한 자료를 제공하되, 추가로:
- 경쟁사 대비 배틀 카드 (자사 제품이 적합하지 않은 시나리오 포함 — 파트너는 정직함을 존중한다)
- 파트너가 잠재 고객에 맞게 커스텀할 수 있는 ROI 계산기
- 잠재 고객이 참고 통화를 요청할 때 제공할 레퍼런스 고객 목록
- 티어별 승인된 할인 권한을 포함한 가격 가이드라인

### 공동 판매 모션

#### 딜 등록

딜 등록은 채널 충돌을 방지하고 파트너가 기회에 시간을 투자할 수 있도록 신뢰를 준다.

**규칙:**
- 파트너가 딜을 등록할 때 제출하는 정보: 회사명, 담당자 정보, 예상 딜 규모, 예상 클로징 일자
- 자사 팀이 48시간 이내에 충돌 여부를 확인 (이미 파이프라인에 있는 어카운트인가?)
- 승인 시: 파트너가 90일간 독점 보호를 받음 (1회 갱신 가능)
- 이미 파이프라인에 있는 어카운트: 파트너가 공동 판매 모션에 참여할 수 있지만, 자사 직접 AE가 주도
- 파트너의 딜이 정체 시 (30일간 활동 없음): 통보 후 보호 해제

#### 매출 분배 모델

| 모델 | 작동 방식 | 적용 시점 |
|:---|:---|:---|
| **추천 수수료** | 파트너가 리드를 추천하고, 자사가 클로징. 파트너는 첫 해 ACV의 10–20%를 받음. | 파트너에게 관계가 있지만 영업 역량이 없을 때 |
| **리셀러 마진** | 파트너가 판매, 고객에게 청구, 자사에게 도매가(보통 정가의 70–80%)를 지급. | 파트너가 고객 관계를 직접 소유하고 싶을 때. 대형 컨설팅 펌에서 흔함. |
| **영향/공동 판매** | 파트너가 자사 AE와 함께 딜에 관여. 파트너는 ACV의 5–15%를 받음. | 파트너가 신뢰도나 구현 전문성을 더하는 복잡한 엔터프라이즈 딜 |
| **마켓플레이스 매출 분배** | 클라우드 마켓플레이스가 3–5% 수수료 (대량 시 협상 가능). 고객은 확정된 클라우드 지출을 사용. | 소진해야 할 클라우드 커밋이 있는 엔터프라이즈 고객 |

### 파트너 포털 요건

최소 기능 파트너 포털:
- 딜 등록 양식 및 상태 추적
- 리드 배분 및 수락 워크플로우
- 교육 모듈 및 인증 추적
- 마케팅 자산 라이브러리 (코브랜드 가능 템플릿)
- 커미션/마진 명세서 및 지급 내역
- 지원 티켓 에스컬레이션 경로
- 성과 대시보드: 파이프라인, 매출, 전환율

처음부터 만들지 말 것. Crossbeam이나 Reveal을 어카운트 매핑에, PartnerStack이나 Allbound를 포털에 사용한다.

### 채널 충돌 처리

채널 충돌은 불가피하다. 미리 대비해야 한다.

| 충돌 유형 | 시나리오 | 해결 방법 |
|:---|:---|:---|
| **직접 영업 vs. 파트너** | 자사 AE가 작업 중인 어카운트를 파트너도 등록 | 먼저 등록한 쪽이 우선 (타임스탬프 확인). 동시 등록이면 파트너와 공동 판매 후 분배. |
| **파트너 vs. 파트너** | 두 파트너가 동일 기회를 주장 | 먼저 등록한 쪽이 우선. 동시 등록이면 고객이 어느 파트너와 일하고 싶은지 선택. |
| **셀프서브 vs. 파트너** | 고객이 PLG를 통해 자사를 발견했지만 파트너가 영향력을 주장 | 파트너 등록 전에 고객이 PLG에서 활성화했으면, 파트너는 전체 추천 수수료가 아닌 축소된 영향 수수료(5%)를 받음. |
| **갱신 충돌** | 파트너가 초기 딜을 가져왔지만 2년간 관여하지 않음. 갱신 커미션을 주장. | 파트너가 어카운트에서 활동을 유지하는 경우에만 갱신 커미션 지급 (활동의 정의: 분기별 고객 체크인, 인증 유지). |

메타 규칙: 충돌 해결 정책을 투명하게 공개하고 파트너 계약에 명시하며 일관되게 적용한다. 파트너는 시스템이 공정하다고 신뢰하면 특정 딜에서 지는 것은 수용한다.

### 시장 개발 펀드 (MDF)

MDF는 파트너에게 공동 마케팅 활동을 위해 제공하는 자금이다. 선물이 아니며 — 공동 투자와 실행 증빙이 필요하다.

**MDF 구조화 방법:**

| 카테고리 | 적격 활동 | 펀딩 모델 | 필요 증빙 |
|:---|:---|:---|:---|
| **수요 창출** | 공동 웨비나, 이벤트, 이메일 캠페인, 디지털 광고 | 50/50 공동 투자 또는 Gold+ 시 100% 펀딩 | 등록/참석자 목록, 리드 결과 |
| **콘텐츠** | 공동 저술 백서, 케이스 스터디, 블로그 포스트 | 100% 펀딩 (자사가 품질 관리) | 게시된 자산, 배포 지표 |
| **이벤트** | 컨퍼런스 스폰서십, 파트너 주최 디너, 로드쇼 | 50/50 공동 투자 | 참석자 목록, 90일 이내 생성된 파이프라인 |
| **인에이블먼트** | 파트너 내부 교육 세션, 해커톤 | 100% 펀딩 | 출석 기록, 인증 완료 현황 |

**MDF 예산 경험 법칙:** 파트너 소싱 매출의 2–5%를 MDF에 재할당한다. MDF 투자 대비 ROI를 추적 — 투자 1달러당 생성 파이프라인이 최소 5–10배여야 한다.

- 단계별 응답률 추적. N단계의 응답률이 1% 미만이면 다시 작성
- 시퀀스 종료 후 분기별 너처링 드립으로 재순환

### 자격 검증 프레임워크

#### BANT (Budget, Authority, Need, Timeline)

BANT는 기본이다. 단순하고 거래형 영업에 효과적이다. 복잡한 엔터프라이즈 딜에서는 MEDDPICC를 추가로 적용한다.

| 기준 | 핵심 질문 | 자격 충족 조건 |
|:---|:---|:---|
| **Budget** | "이를 위한 예산이 배정되어 있나요?" / "이 정도 규모의 투자에 대한 일반적인 승인 절차는 어떻게 되나요?" | 예산이 존재하거나 만들 수 있음. 잠재 고객이 조달 프로세스를 설명할 수 있음. |
| **Authority** | "이 결정에 다른 누가 관여하나요?" / "팀에서 유사한 구매 결정을 어떻게 내렸는지 과정을 설명해 주세요." | 의사결정권자를 파악했고 접근 가능함. 의사결정 프로세스가 명확함. |
| **Need** | "평가를 진행하게 된 계기가 무엇인가요?" / "이 문제를 해결하지 않으면 어떻게 되나요?" | 측정 가능한 영향이 있는 실제 문제가 있음. 현상 유지에 비용이 따름. |
| **Timeline** | "언제까지 도입이 필요하나요?" / "일정을 주도하는 이벤트가 있나요?" | 강제 요인이 있음 — 계약 갱신, 이사회 지시, 컴플라이언스 기한, 경쟁 압력. |

#### MEDDPICC (엔터프라이즈 및 복잡한 딜용)

| 요소 | 의미 | 디스커버리 질문 | 위험 신호 |
|:---|:---|:---|:---|
| **Metrics** | 구매자가 필요로 하는 정량적 비즈니스 성과 | "어떤 KPI에 영향을 미치나요? 얼마나?" | 구매자가 측정 가능한 성과를 명확히 설명하지 못함 |
| **Economic Buyer** | 결제 승인 권한자 | "최종 예산 권한은 누구에게 있나요? 만나봤나요?" | EB와 대화한 적 없이 챔피언하고만 소통 |
| **Decision Process** | 평가 및 결정 방식 | "여기서 계약 체결까지의 단계를 설명해 주세요." | "알아서 하겠습니다" — 정의된 프로세스 없음 = 실제 딜이 아님 |
| **Decision Criteria** | 평가 기준 | "필수 요건과 우대 사항은 무엇인가요?" | 평가 기준이 경쟁사 기능 세트와 정확히 일치 |
| **Paper Process** | 법무, 조달, 보안 검토 | "조달 프로세스는 어떤가요? 누가 계약을 검토하나요?" | 모르거나 논의를 거부 — 법무 단계에서 딜이 정체될 것 |
| **Identified Pain** | 구매를 추진하는 구체적 문제 | "현 상황의 비용은 얼마인가요? 누가 가장 크게 느끼나요?" | 고통이 이론적일 뿐, 예산 권한자가 체감하지 않음 |
| **Champion** | 자사가 없는 자리에서 내부에서 홍보해주는 내부 옹호자 | "이 이니셔티브를 내부에서 후원하는 사람은 누구인가요?" | "챔피언"이 EB와의 미팅을 잡아주지 못함 |
| **Competition** | 평가 중인 다른 대안 | "어떤 다른 접근 방식을 검토하고 계신가요, 아무것도 하지 않는 것 포함해서?" | 말해주지 않거나, 이미 경쟁사 PoC에 깊이 참여 중 |

**MEDDPICC 스코어링:** 각 요소를 0 (미확인), 1 (부분 확인), 2 (완전 확인)로 점수화. 10/16 미만인 딜은 커밋 포캐스트에 포함하지 말아야 한다.

### 디스커버리 콜 프레임워크

디스커버리 콜은 데모가 아니다. 대화 시간의 30% 미만만 말해야 한다.

**구성 (30분):**

1. **라포 및 안건** (3분): 기대치 설정. "무언가를 보여드리기 전에 상황을 먼저 이해하고 싶습니다. 몇 가지 질문을 드려도 될까요?"
2. **현재 상태** (8분): 현재 어떻게 업무를 처리하는지. 어떤 도구를 사용하는지. 워크플로우가 어떤 형태인지.
3. **문제 및 페인** (8분): 무엇이 문제인지. 무엇이 불편한지. 비즈니스 영향은 무엇인지. 모든 것을 수치화.
4. **원하는 미래 상태** (5분): 이상적인 상태는 어떤 모습인가? 이 문제가 해결되면 무엇이 달라지는가?
5. **프로세스 및 타임라인** (4분): 의사결정 프로세스, 이해관계자, 타임라인, 예산.
6. **다음 단계** (2분): 구체적 일정과 구체적 행동에 합의. "목요일 오후 2시에 팀 리드와 기술 심층 논의를 잡아볼 수 있을까요?"

**규칙:**
- 오픈형 질문을 한다. "이 문제가 있으신가요?"라고 묻지 않는다 (예의상 그렇다고 대답할 것이다)
- 눈에 보이게 메모한다 — 경청하고 있음을 보여주고, 후속 조치를 위한 자료가 된다
- 이해한 내용을 되짚어 확인한다: "제가 올바르게 이해했다면, 핵심 문제는..."
- 디스커버리 중에 데모를 하지 않는다. 명시적으로 요청해도 5분 이내로 제한.

### 데모 베스트 프랙티스

- **고객의 페인에 맞춤**: 디스커버리에서 세 가지 문제가 드러났으면, 정확히 그 세 가지 솔루션만 데모. 그 외에는 아무것도.
- **"와우" 모먼트로 시작**: 서서히 쌓아가지 않는다. 가장 인상적인 것을 처음 2분에 보여줌.
- **가능하면 고객의 실제 데이터 사용**: 잠재 고객의 실제 데이터를 사용한 데모는 일반적인 데모보다 3배 효과적.
- **20분 이내로 유지**: 20분 이후 집중도가 급격히 떨어진다. Q&A 시간을 확보.
- **핵심 기능마다 멈추기**: "현재 하시는 방식과 비교하면 어떤가요?" 참여를 유도하고 반론을 일찍 파악.
- **녹화 후 공유**: 녹화본을 보내준다. 챔피언이 내부에서 설득할 때 사용할 것이다.

### 엔터프라이즈 멀티스레딩

여러 핵심 역할과 동시에 관계를 구축한다. 단일 스레드 딜(한 사람만 아는 경우)은 수주율이 30–40% 낮다.

| 이해관계자 | 의사결정에서의 역할 | 관심사 | 접근 방법 |
|:---|:---|:---|:---|
| **챔피언** | 내부 옹호자, 자사가 없는 자리에서 설득 | 본인의 성과를 돋보이게 하는 것, 팀의 문제 해결 | 자료로 무장시키고, 히어로로 만들어 줌 |
| **Economic Buyer** | 최종 결제 승인 (보통 VP+ 또는 C레벨) | ROI, 전략적 정합성, 리스크 완화 | 경영진 브리핑, ROI 모델, 동종 업계 레퍼런스 |
| **Technical Buyer** | 기술 적합성 평가 (보통 IT, 엔지니어링, 보안) | 연동, 보안, 확장성, 유지보수 부담 | SE 주도 기술 심층 논의, 아키텍처 리뷰, 보안 질의서 |
| **최종 사용자** | 실제로 매일 제품을 사용할 사람 | 사용 편의성, 워크플로우 적합성, 업무를 더 어렵게 만들지 않는지 | 핸즈온 트라이얼, 파일럿 프로그램, 사용자 피드백 세션 |
| **조달** | 구매 프로세스 관리 | 컴플라이언스, 가격 벤치마킹, 계약 조건 | 요건을 미리 작성하여 제공, 서류 작업에 신속 대응 |
| **법무** | 계약 검토 | 데이터 처리, 책임, SLA 조건 | 레드라인 준비된 계약서, DPA, 보안 인증서 제공 |

**멀티스레딩 전술:**
- 챔피언에게 다른 이해관계자를 소개해 달라고 요청
- 더 넓은 팀에 노출될 수 있는 "런치 앤 런" 또는 워크숍 개최
- 이해관계자별 다른 콘텐츠 전송 (Technical Buyer에게 기술 콘텐츠, Economic Buyer에게 ROI)
- 챔피언이 퇴사하면 다른 스레드가 없으면 딜이 소멸

### 협상 전술

**원칙:**
- 스스로와 협상하지 않는다. 할인을 요청하면 대가를 물어본다 (장기 계약, 케이스 스터디, 더 큰 범위, 더 빠른 서명).
- 높게 앵커링한다. 첫 제안은 정가여야 한다. 정가에서 할인하는 것은 양보이고, 정가 이하에서 시작하면 돈을 남기는 것이다.
- 시간 압박은 양방향으로 작동한다. 상대에게 기한이 있다면, 자사에게도 있다. "이 가격은 분기 말까지 유효합니다."
- 워크어웨이 파워가 진짜 파워다. 떠날 수 없으면 협상할 수 없다.

**일반적인 할인 구조:**

| 상대가 원하는 양보 | 대가로 요청할 것 |
|:---|:---|
| 10–15% 할인 | 연간 선불 (월/분기별 대신) |
| 15–25% 할인 | 다년 계약 (2–3년) |
| 연장 파일럿 | 정의된 성공 기준 및 충족 시 자동 전환 |
| 추가 시트 무료 제공 | 6개월 이내 전체 부서 확장 약속 |
| 커스텀 결제 조건 | 공개 케이스 스터디 + 로고 사용권 |
| 기능 요청 | 공동 개발 계약 + 코마케팅 권한 |

**엔터프라이즈 가격 전략:**
- 항상 월간이 아닌 연간 가격으로 견적 (더 높게 앵커링)
- 세 가지 티어를 만들 것; 대부분의 구매자는 중간을 선택
- 구현/교육을 계약에 번들 (ACV 증가, 부실 도입 리스크 감소)
- 매우 큰 딜의 경우: 사용량 기반 또는 소비 기반 가격으로 구매자의 리스크를 줄이면서 자사 인센티브를 고객 성장에 맞춤

### 계약 협상 플레이북

| 단계 | 자사 목표 | 일반적 반론 | 대응 |
|:---|:---|:---|:---|
| **MSA/약관** | 표준 조건, 최소한의 레드라인 | "저희 법무팀이 XYZ를 요구합니다" | 일반적인 엔터프라이즈 조건을 미리 포함한 레드라인 준비 계약서를 구축. AE가 법무 검토 없이 제안할 수 있는 "사전 승인 양보" 목록을 준비. |
| **SLA** | 달성 가능한 수준을 약속 | "99.99% 가동률이 필요합니다" | 실제 가동률 실적을 보여준다. 정의된 크레딧과 함께 99.9%를 제안. 99.99% = 연간 52분 다운타임 — 달성할 수 있을 때만 약속. |
| **데이터 처리** | 표준 DPA | "데이터가 [지역]에 있어야 합니다" | 지원 가능하면 좋고. 불가능하면 솔직히 말하고 암호화 + 접근 제어를 대안으로 제시. 달성 불가한 것을 약속하지 않음. |
| **책임** | 책임 한도 설정 | "데이터 유출에 대한 무제한 책임" | 지급 수수료 12개월분으로 한도 설정. 사이버 보험에 가입하고 증서를 공유. 대부분의 법무팀이 수용함. |
| **해지** | 연간 최소, 비례 환불 | "언제든 편의 해지를 원합니다" | 90일 사전 통지와 미사용 선불 기간의 비례 환불을 제안. 연간 계약의 즉시 해지에는 절대 동의하지 않음. |
| **자동 갱신** | 30일 옵트아웃 통지와 함께 자동 갱신 | "자동 갱신 안 됩니다" | 수용한다 — 불만족 고객에게 자동 갱신을 강제하면 어차피 이탈이 발생한다. 기간 종료 90일 전에 통지. |

## 5. 콘텐츠 주도 성장

콘텐츠 주도 성장은 "블로그를 운영하는 것"이 아니다. 퍼널의 모든 단계에서 수요를 포착하고 시간이 지남에 따라 복리로 성장하는 체계적인 콘텐츠 운영을 구축하는 것이다. B2B SaaS에서 콘텐츠는 1년 이후 가장 높은 ROI를 내는 채널인 경우가 많은데, 콘텐츠는 게시된 후에도 계속 트래픽과 리드를 생성하기 때문이다.

### B2B SaaS를 위한 SEO 전략

#### 검색 의도별 키워드 전략

| 의도 유형 | 키워드 예시 | 콘텐츠 유형 | 전환 기대치 |
|:---|:---|:---|:---|
| **탐색형** | "[제품명] 로그인", "[제품] 가격" | 제품 페이지, 가격 페이지 | 높음 (이미 사용자 또는 평가자) |
| **상업적 조사** | "최고의 [카테고리] 소프트웨어", "[경쟁사] 대안", "[제품 A] vs [제품 B]" | 비교 페이지, 대안 페이지, 리뷰 모음 | 높음 (활발한 구매 결정 중) |
| **정보형 (문제 인식)** | "[자사 제품이 해결하는 문제]를 해결하는 방법" | 하우투 가이드, 튜토리얼, 템플릿 | 중간 (문제 인식 단계에서 수요 포착) |
| **정보형 (미인식)** | 넓은 산업 주제 | 사고 리더십, 리서치 리포트 | 낮음 (퍼널 상단, 브랜드 구축) |

**상업적 조사 키워드를 먼저 우선순위에 놓을 것.** 정보형 콘텐츠보다 5–10배 높은 비율로 전환되며, "경쟁적"이거나 "영업적"으로 느껴져서 간과되는 경우가 많다. 잘 만든 "[경쟁사] 대안" 페이지는 가장 높은 전환율의 오가닉 페이지가 될 수 있다.

#### B2B를 위한 테크니컬 SEO 체크리스트

- 연동, 사용 사례, 템플릿을 위한 프로그래매틱 페이지 (오가닉으로 확장)
- 스키마 마크업: FAQ, HowTo, SoftwareApplication, Organization
- Core Web Vitals: LCP 2.5초 미만, CLS 0.1 미만
- 내부 링크: 모든 블로그 포스트가 최소 2개의 제품 페이지와 2개의 관련 블로그 포스트에 링크
- 중복/유사 콘텐츠에 대한 Canonical 태그
- 콘텐츠 유형별 분리된 XML 사이트맵 (블로그, 문서, 제품 페이지)
- 다국어 콘텐츠가 있을 경우 Hreflang 태그

### 퍼널 단계별 콘텐츠 유형

| 퍼널 단계 | 콘텐츠 유형 | 목표 | 배포 채널 |
|:---|:---|:---|:---|
| **퍼널 상단** | 산업 리포트, 오리지널 리서치, 넓은 하우투 가이드, 팟캐스트 | 브랜드 인지도, 이메일 수집 | SEO, 소셜 미디어, 유료 소셜 (LinkedIn) |
| **퍼널 중단** | 비교 페이지, 사용 사례 심층 분석, 웨비나, 템플릿, 계산기 | 제품 평가 및 고려 | SEO, 이메일 너처링, 리타겟팅 |
| **퍼널 하단** | 케이스 스터디, ROI 계산기, 데모 비디오, 보안/컴플라이언스 문서 | 반론 제거, 구매 지원 | 영업 인에이블먼트, 이메일 너처링, 웹사이트 |
| **구매 후** | 문서, 아카데미 코스, 커뮤니티 콘텐츠, 체인지로그 | 도입, 확장, 옹호 | 인앱, 이메일, 커뮤니티 플랫폼 |

### 개발자 마케팅 및 기술 콘텐츠

바이어가 개발자이거나 기술 사용자라면, 일반적인 마케팅 콘텐츠는 오히려 해가 된다. 개발자는 마케팅 냄새를 멀리서도 맡으며 즉시 제품을 무시할 것이다.

**개발자에게 효과적인 것:**
- **실제로 좋은 문서**: 이것이 #1 마케팅 자산이다. 명확하고, 완전하며, 작동하는 코드 예제가 있어야 한다. 문서가 나쁘면 다른 아무것도 소용없다.
- **인터랙티브 튜토리얼**: 샌드박스나 플레이그라운드를 임베드. 가입 없이 API를 시도할 수 있게 한다.
- **오픈소스 기여**: 자사 제품에 인접한 라이브러리, 도구, 또는 프레임워크를 오픈소스로 공개. 호감과 배포력을 구축.
- **엔지니어가 작성한 기술 블로그 포스트**: 실제 엔지니어가 해결한 실제 문제에 대해 작성. 대필된 마케팅 포장 글이 아닌.
- **CLI 도구 및 SDK**: 연동을 마찰 없이 만든다. 시작하려고 20페이지짜리 PDF를 읽어야 하면 이미 진 것이다.
- **커뮤니티**: 개발자가 서로 돕는 Discord, Slack 커뮤니티 또는 포럼. 커뮤니티는 방어벽이다.

**개발자에게 효과 없는 것:**
- 게이트 콘텐츠 (개발자는 그냥 떠남)
- "데모 요청"만 있는 CTA (개발자는 보는 것이 아닌 직접 시도하고 싶어 함)
- 마케팅 용어로 작성된 케이스 스터디 ("시너지", "레버리지", "트랜스폼")
- 숨겨지거나 "영업팀에 문의"인 가격 (개발자는 "비싸고 귀찮다"고 추정)

### 실제로 효과 있는 사고 리더십

사고 리더십은 모든 사람이 말하는 것을 재작성하는 것이 아니다. 증거에 기반한 반대 의견이나 새로운 관점을 갖는 것이다.

**사고 리더십 콘텐츠 프레임워크:**
1. **해당 산업에서 널리 받아들여지는 믿음을 식별**하되, 동의하지 않거나 뉘앙스를 더할 수 있는 것
2. **반대 견해를 데이터로 뒷받침**: 자사 제품 데이터, 고객 데이터, 오리지널 리서치, 또는 신뢰할 수 있는 제3자 데이터
3. **실행 가능하게 만들기**: 단순히 반대 의견만 제시하지 않는다. 무엇을 다르게 해야 하는지 알려준다.
4. **실명 인물이 게시**: 브랜드 계정의 사고 리더십은 일반 콘텐츠다. 실명의 창업자/경영진의 사고 리더십은 기업에 도움이 되는 개인 브랜드를 구축한다.

**사고 리더십 배포:**
- LinkedIn 포스트 (B2B에서는 800–1200단어 포스트가 최적, 주 2–3회 게시)
- 산업 컨퍼런스 발표 (발표 신청, 또는 자체 이벤트 개최)
- 팟캐스트 게스트 출연 (자사 ICP가 듣는 팟캐스트 발굴)
- 산업 매체에 배포

## 6. 프로덕트 레드 세일즈

프로덕트 레드 세일즈(PLS)는 PLG와 다르다. PLG는 셀프서브 고객 확보에 관한 것이다. PLS는 제품 사용 데이터를 활용하여 기존 어카운트에서 확장 매출을 견인하는 것이다. 진짜 돈은 여기에 있다 — 120% 이상의 순매출유지율은 PLS에서 나온다.

### 사용량 기반 확장 플레이북

#### 확장 트리거

| 트리거 | 시그널 | 담당자 | 조치 |
|:---|:---|:---|:---|
| **시트 한도 근접** | 라이선스 시트의 80% 이상 사용 중 | CSM | 사전 아웃리치: "팀이 성장하고 있네요 — 필요한 모든 분이 액세스할 수 있도록 하겠습니다." |
| **사용량 한도 근접** | API 호출, 스토리지, 워크플로우 등의 80% 이상 사용 | 자동화 + CSM | 80%에 인앱 경고, 90%에 CSM 아웃리치. 업그레이드 경로는 원클릭이어야 함. |
| **새 팀/부서 도입** | 새 이메일 그룹이나 비용 센터의 사용자 출현 | AE | 확장 기회: 새로운 사업부. 해당 부서의 특정 니즈에 대한 새 디스커버리 진행. |
| **기능 게이트 도달** | 사용자가 유료 기능 게이트에 반복적으로 접근 | 자동화 | 인앱: 해당 기능의 역할을 보여주고, 프리미엄 기능 7일 트라이얼 제안. |
| **파워유저 출현** | 개별 사용자가 평균 대비 3배 참여도 | CSM | 내부 챔피언으로 영입. 고객 자문위원회에 초대. 다른 팀의 도입을 이끌도록 활용. |
| **연동 확장** | 어카운트가 API를 통해 새 서드파티 도구 연결 | SE + CSM | 프리미엄 연동 지원, 엔터프라이즈 연동 티어 논의. |

#### 시트 확장 전략

시트 확장은 가장 쉬운 확장 모션이다. 가치 제안이 자명하기 때문이다: 더 많은 사람에게 액세스가 필요하다.

**시트 확장을 유도하는 바이럴 루프:**
- **계정이 필요한 협업 기능**: 문서, 대시보드, 또는 워크스페이스를 공유하려면 수신자가 가입해야 함
- **@멘션 및 알림**: 비사용자를 댓글에서 멘션하면 이메일 초대 발송
- **팀 템플릿 및 공유 리소스**: 신규 입사자가 팀 템플릿에 액세스해야 하므로 자연스러운 확장 발생
- **권한 관리**: 관리자 기능(감사 로그, 역할 기반 접근 제어)은 관리자 기능이 포함된 플랜으로 업그레이드 필요

**시트 확장 가격:**
- 볼륨 할인을 너무 일찍 제공하지 않는다 — 처음 50시트는 정가로
- 자연스러운 구간에서 볼륨 할인 제공: 50, 100, 250, 500+ 시트
- "플랫폼 요금 + 시트당 요금" 모델을 고려하여 매출 하한선 유지
- 시트 추가에 대한 연간 커밋 + 분기별 정산 (고객이 분기 중간에 시트를 추가하면 즉시 비례 지급, 갱신 시 정산)

### 인앱 업그레이드 프롬프트

**업그레이드 프롬프트 배치 위치 (전환율 순):**

1. **가치 전달 순간**: 사용자가 작업을 성공적으로 완료 → "무료 [항목] 10개 중 8개를 사용하셨습니다. 무제한으로 업그레이드하세요." (최고 전환율 — 사용자가 방금 가치를 경험)
2. **기능 게이트**: 사용자가 프리미엄 기능 클릭 → 미리보기로 기능 설명, 트라이얼 또는 업그레이드 제안 (높은 전환율 — 의도 표현됨)
3. **사용량 마일스톤**: "[제품]으로 이번 달 40시간을 절약하셨습니다. Pro 플랜이면 팀 전체가 200시간 이상 절약할 수 있습니다." (중간 전환율 — 소셜 프루프 + 추정)
4. **설정/관리 페이지**: 관리자가 워크스페이스를 관리할 때 → 관리자 기능 업셀 (중간 전환율 — 관리 컨텍스트)
5. **대시보드/홈페이지**: 지속적 배너 → 피할 것. 배너 무시 현상이 빠르게 발생. (낮은 전환율 — 맥락적 관련성 없음)

**안티패턴:**
- 모달 팝업으로 워크플로우를 방해하지 않는다
- 구매 결정권이 없는 사용자에게 업그레이드 프롬프트를 보여주지 않는다 (관리자/결제 담당자를 식별)
- 사용자가 이미 사용 중인 기능을 게이트하지 않는다 (업그레이드가 아닌 분노를 유발)
- 세션당 동일한 업그레이드 프롬프트를 두 번 이상 보여주지 않는다

### 사용량 제한 설계

사용량 제한을 어떻게 설계하느냐가 다른 거의 모든 제품 결정보다 무료→유료 전환율에 더 큰 영향을 미친다.

**좋은 제한**은 비즈니스 가치에 비례하여 확장되는 것을 제한한다:
- 활성 프로젝트 또는 워크플로우 수
- 팀원 (협업)
- 연결된 연동
- 데이터 보존 기간
- API 호출량
- 스토리지

**나쁜 제한**은 징벌적으로 느껴지는 것을 제한한다:
- 로그인 횟수
- 해당 카테고리에서 기본인 기능
- 사용자 본인 데이터의 내보내기
- 경고 없이 일정 기간 후 핵심 기능 제한

**제한의 80/20 법칙:** 무료 티어는 기능의 80%를 제공하되 규모의 20%만 제공해야 한다. 사용자가 모든 것을 할 수 있어야 하지만 (가치 증명), 실제 업무에 필요한 볼륨으로는 사용할 수 없어야 한다.

## 7. 해외 확장

### 시장 선정 기준

해외 확장은 멋져 보여서 하는 것이 아니다. 데이터가 수요가 있다고 말할 때 하는 것이다.

#### 시장 스코어링 매트릭스

| 기준 | 가중치 | 측정 방법 |
|:---|:---|:---|
| **기존 오가닉 수요** | 30% | 해당 국가에서의 웹사이트 트래픽, 인바운드 문의, 기존 고객 |
| **시장 규모 (TAM)** | 20% | 해당 시장에서 자사 ICP에 해당하는 기업 수 (LinkedIn Sales Nav, Statista, 현지 산업 리포트 활용) |
| **경쟁 환경** | 15% | 현지 경쟁사가 지배적인가? 시장이 그린필드인가 레드오션인가? |
| **비즈니스 용이성** | 15% | 결제 인프라, 법적 복잡성, 언어 장벽, 시간대 겹침 |
| **지불 의향** | 10% | 해당 시장의 평균 SaaS 지출, 가격 민감도, 외화 안정성 |
| **전략적 가치** | 10% | 이 시장을 확보하면 인접 시장이 열리는가? (예: 영국 → 유럽, 싱가포르 → 동남아시아) |

**미국 기반 B2B SaaS의 일반적 확장 순서:**
1. 영국 및 캐나다 (영어권, 유사한 비즈니스 문화, 최소한의 현지화)
2. 호주 및 서유럽 (독일, 프랑스, 네덜란드, 북유럽)
3. 일본 및 동남아시아 (싱가포르를 허브로)
4. 라틴 아메리카 (브라질 및 멕시코)
5. 중동 및 아프리카 (UAE를 허브로)

### 현지화 전략

현지화는 번역이 아니다. 현지 기대에 맞게 제품, 마케팅, 가격, GTM을 적응시키는 것이다.

#### 현지화 티어

| 티어 | 현지화 대상 | 투자 규모 | 시기 |
|:---|:---|:---|:---|
| **티어 1: 최소 기능** | 현지 언어 웹사이트, 현지 통화 가격, 현지 결제 수단, 현지 업무 시간 기본 지원 | $50K–$100K | 해당 시장에 유료 고객이 20명 이상 있을 때 (오가닉) |
| **티어 2: 본격** | 제품 UI 번역, 현지 케이스 스터디, 현지 콘텐츠/SEO, 현지 SDR 또는 파트너, 현지 도메인 (예: .de, .co.jp) | $200K–$500K | 해당 시장이 매출의 5% 이상을 차지할 때 |
| **티어 3: 풀** | 현지 법인, 현지 팀 (영업 + CS + 마케팅), 현지 데이터 레지던시, 완전한 제품 현지화 (날짜/시간/숫자 형식, RTL 지원 포함), 현지 파트너십 | $1M+ | 해당 시장이 매출의 15% 이상을 차지하거나 전략적 우선순위일 때 |

### 지역별 컴플라이언스 고려사항

| 지역 | 주요 규정 | 자사에 미치는 영향 |
|:---|:---|:---|
| **EU/EEA** | GDPR, ePrivacy Directive, Digital Services Act | 데이터 처리 계약, 쿠키 동의, 삭제권, 데이터 이동권, DPO 선임 필요 가능, EU 데이터 레지던시 요구 가능 |
| **영국** | UK GDPR (브렉시트 이후), Data Protection Act 2018 | EU GDPR과 유사하지만 별도 체제. 영국 전용 개인정보 정책 및 잠재적으로 영국 대리인이 필요. |
| **독일** | GDPR + 더 엄격한 현지 해석, 직장평의회, Handelsgesetzbuch | 독일 법원이 GDPR을 더 엄격하게 해석. 직원 활동을 모니터링하는 소프트웨어에 대해 직장평의회 승인 필요 가능. 인보이스 요건이 구체적. |
| **일본** | APPI (개인정보보호법) | 국경간 데이터 이전 제한 (기본적으로 적정성 판단 또는 동의 필요). 개인정보 정책이 일본어여야 함. |
| **브라질** | LGPD (일반 데이터 보호법) | GDPR과 유사한 프레임워크. 현지 DPO에 해당하는 역할 필요. 포르투갈어 개인정보 공개. |
| **인도** | DPDP Act 2023 | 특정 카테고리에 대한 데이터 현지화 요건. 동의 관리. 아직 발전 중 — 새 규칙 모니터링 필요. |
| **중동 (UAE/KSA)** | UAE PDPL, KSA PDPL | 비교적 새로운 데이터 보호법. 일부 분야에서 데이터 현지화 요건. 아랍어 공개가 필요할 수 있음. |
| **호주** | Privacy Act 1988, APPs | 국경간 공개 요건. 통지 가능 데이터 유출 체계. GDPR보다 덜 엄격하지만 실질적 집행. |
| **캐나다** | PIPEDA (연방), Quebec Law 25 | Quebec의 Law 25는 GDPR과 유사. Quebec에서 프랑스어 요건. 특정 처리에 대한 개인정보 영향 평가. |

### 해외 확장을 위한 현지 파트너십

대부분의 비영어권 시장에서 현지 파트너는 매출까지의 가장 빠른 경로다. 파트너가 제공하는 것:
- 언어 및 문화적 유창성
- 자사 ICP 내 기존 고객 관계
- 현지 조달 프로세스에 대한 지식
- 시장에 신규 진입하는 외국 벤더에게 없는 신뢰도

**영입할 현지 파트너 유형:**

| 파트너 유형 | 역할 | 발굴 방법 |
|:---|:---|:---|
| **현지 시스템 인테그레이터** | 구현, 커스터마이징, 리셀 | 산업 컨퍼런스, LinkedIn, 기존 고객에게 현지에서 누구와 일하는지 질문 |
| **현지 컨설팅 펌** | 자문, 변경 관리, 리셀 | Big 4 현지 오피스, 자사 카테고리 전문 부티크 컨설턴트 |
| **현지 기술 디스트리뷰터** | 빌링, 지원, 리셀 (현지 결제, 인보이스, 지원 처리) | 클라우드 마켓플레이스 프로그램, 디스트리뷰터 네트워크 (Ingram Micro, TD Synnex) |
| **현지 ISV** | 자사 제품을 현지 솔루션에 통합 | 타겟 고객이 이미 사용 중인 현지 제품 확인 |

**해외 파트너십 딜 구조:**
- 국내보다 높은 마진 (보통 25–35% vs. 15–25%), 시장 개발 노력에 대한 보상
- 국가 전체 독점이 아닌, 특정 버티컬이나 국가 내 지역에 대해서만 독점
- 연간 검토를 포함한 최소 매출 커미트먼트
- 현지 마케팅에 대한 공동 투자 (국내보다 높은 비율로 MDF 펀딩)
